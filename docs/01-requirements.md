# 요구사항 정의

## 프로젝트 개요

개인이 보유한 주식의 거래내역을 입력하면, 평균 매입가와 실현/평가손익, 수익률을
자동으로 계산하고 일간 스냅샷으로 수익률 추이를 확인할 수 있는 백엔드 API

직접 보유한 종목을 거래내역만 넣으면 평가손익과 수익률 추이를 알아서 계산해주는 도구를 만들고자 시작한 프로젝트

## 범위

### 포함
- 단일 사용자 (별도 인증 없음)
- KRX 한국 주식
- 외부 시세 API: 공공데이터포털 - 금융위원회 주식시세정보

### 제외
| 항목 | 사유 |
| --- | --- |
| 해외 주식, 환율 | 단일 시장에서 도메인 로직을 충분히 다루기 위함 |
| 실시간 호가 / 주문 체결 | 분석·관리 도구 성격이며 트레이딩 시스템이 아님 |
| 증권사 계좌 연동 | 인증·보안 요구사항이 본 프로젝트의 핵심 가치와 거리가 있음 |
| 다중 사용자 | 본인용 도구로 시작하며, 추후 확장 여지로 남김 |

## 기능 요구사항

| ID | 기능 | 설명 |
| --- | --- | --- |
| FR-01 | 거래내역 등록 | 매수/매도 거래 입력 (종목코드, 수량, 단가, 거래일) |
| FR-02 | 거래내역 조회/수정/삭제 | 거래내역 CRUD |
| FR-03 | 평균 매입가 자동 계산 | 매수 시 이동평균법으로 갱신, 매도 시 유지 |
| FR-04 | 보유 종목 자동 계산 | 매수/매도 누적 수량 기반 |
| FR-05 | 실현손익 계산 | 매도 시점 (매도가 - 평균매입가) × 수량 |
| FR-06 | 평가손익 계산 | (현재가 - 평균매입가) × 보유수량 |
| FR-07 | 현재가 조회 | 외부 시세 API 호출 + Redis 캐시 (TTL 5분) |
| FR-08 | 포트폴리오 조회 | 총자산, 총평가손익, 총실현손익, 총수익률 |
| FR-09 | 일간 포트폴리오 스냅샷 | 매일 자정 Spring Scheduler로 저장 |
| FR-10 | 수익률 추이 조회 | 최근 30일 / 90일 시계열 |

## 비기능 요구사항

| 항목 | 목표 |
| --- | --- |
| 응답 시간 | 현재가 조회 P95 < 100ms (캐시 적중 시) |
| 외부 API 부하 | 캐시 적용 후 호출량 80% 이상 감소 |
| 데이터 정합성 | 동시 매수/매도 시 평균 매입가 정확성 보장 |
| 가용성 | 외부 API 장애 시 stale 캐시 데이터로 폴백 |
| 운영성 | 환경별 설정 분리, 로컬에서 단일 명령으로 실행 |
| 환경 일관성 | 개발·배포 환경을 Linux(Ubuntu)로 통일하여 OS 차이로 인한 이슈 차단 |

## 외부 API

- 제공기관: 금융위원회
- 서비스명: 주식시세정보 (공공데이터포털)
- 인증: API Key
- 응답: 일별 시세 (종가 기준)
- 호출 제한 존재 → Redis 캐시로 호출량 제어
